Кайманов Константин 1-группа 2021год


Проект: пошаговая стратегия с простейшим интерфейсом

Инструменты: PyQt, при использовании некоторых дополнительных библиотек python. База данных SQLlite

Описание игры: В процессе учатсвуют два игрока, находящиеся за одним компьютером, игроки будут сменяться по принципу передачи хода.
 Процесс игры: Классическая стратегия. На поле 8х8 случайным образом располагаются предметы ландшафта: в начале игры: горы, поля и замок каждого игрока
   Замок игрока распологается случайным образом в начале его половины (4х2 с каждой стороны) горы и поля располагаются по принципу: по одному полю на начальной территории
   Горы и большая часть полей располагаются на игровой зоне(квадрат в середине 4х4) для победы нужно захватить замок противника
   Естественно, для этого нужны воины, их будет предположительно 2-3 типа, в замке можно будет нанимать воинов, предположительно делать
   еще какие-то странные действия. Кликнув по клетке(пустая клетка, поле) можно будет оптравить в нее войска для захвата, или просто постоять, подумать
   для захвата клетки нужно, чтобы минуло два хода(возможно кл будет изменено) Противник может послать свою армию для отхвата поля, тогда все решит битва
   процесс битвы будет осуществляться по количеству войск, их типу, баффам конкретной клетки(пример: поле не имеет дебаффов, зато если его потоптать
   доход снизится) поля приносят доход, за который можно нанимать войска. Каждому игроку в начале дается определенное кол-во монет.
   (скорее всего описание будет меняться, что-то будет добавляться, что-то убираться(чаще всего 2)).
 Некоторые особенности интерфейса: свои действия и действия противника можно будет посмотреть в логе, возмножно будут юниты-пасхалки,
   которых можно будет встретить на карте.
